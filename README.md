## 1. 플랫폼 비전과 철학

### 1-1. 오거서(五去書)란

오거서(五去書)는 **"다섯 가지 방법으로 책에 다가간다"** 는 뜻을 담은 성균관대학교 학술정보관의 독서 진흥 플랫폼입니다.

| 항목 | 내용 |
|------|------|
| 미션 | 대학생의 독서 습관 형성과 독서 공동체 활성화 |
| 핵심 가치 | 혼자가 아닌 **함께 읽는 경험**, 읽기를 넘어 **표현하는 독서** |
| 대상 사용자 | 성균관대학교 재학생 · 교직원 · 동문 |
| 운영 주체 | 학술정보관 독서문화팀 |

### 1-2. 왜 "오거서"인가 — 문제의식

대학 독서 문화의 현실:

| 문제 | 현상 | 영향 |
|------|------|------|
| **독서의 고립화** | 책을 읽는 행위가 개인에 머무름. 읽은 후 공유·토론할 공간이 없음 | 독서 동기 약화, 깊이 있는 이해 부족 |
| **독서 경험의 단일화** | "눈으로 읽기"에 한정. 소리 내어 읽기, 따라 쓰기 등 다양한 독서법이 실천되지 않음 | 독서가 의무적·수동적 활동으로 인식됨 |
| **디지털 세대의 감각 단절** | SNS·유튜브 등 시각·청각 자극에 익숙하지만, 정작 책과의 감각적 연결이 끊어져 있음 | "책은 재미없다"는 고정관념 고착 |
| **공동체 독서 인프라 부재** | 독서 동아리가 있어도 모집·운영·아카이빙 도구가 분산됨 | 지속성 부족, 참여 이탈 |

오거서는 이 네 가지 문제를 해결하기 위해, 독서 행위를 **5가지 방법(존)**으로 분해하고, 각 방법에 최적화된 디지털 경험을 설계했습니다.

### 1-3. 핵심 철학: "책에 다가가는 다섯 가지 길"

오거서의 "다섯 가지 방법"은 단순한 기능 분류가 아닙니다. **독서라는 행위를 몸 전체로 확장하는 철학적 프레임**입니다.

```
                   ┌─────────────┐
                   │   독서(讀書)  │
                   │  책을 읽는다  │
                   └──────┬──────┘
                          │
          ┌───────┬───────┼───────┬───────┐
          ▼       ▼       ▼       ▼       ▼
      ┌──────┐┌──────┐┌──────┐┌──────┐┌──────┐
      │ 함께 ││ 소리 ││ 감정 ││ 따라 ││ 깊이 │
      │읽는다││내어  ││으로  ││쓰며  ││음미  │
      │      ││읽는다││만난다││새긴다││한다  │
      │북클럽││ 낭독 ││ 듣기 ││ 필사 ││ 서평 │
      └──────┘└──────┘└──────┘└──────┘└──────┘
        共       聽       聽       觸       味
```

각 방법은 독자에게 서로 다른 감각을 일깨워, 같은 책도 **다섯 가지 다른 경험**으로 만날 수 있게 합니다.

## 2. 오감도서관 컨셉 정의

### 2-1. 오감(五感) 프레임

오거서의 "다섯 가지 방법으로 책에 다가간다"는 철학을 **오감(五感)** 프레임으로 확장하면, 독서 행위가 단순한 정보 소비가 아닌 **온몸으로 책을 체험하는 감각적 경험**이 됩니다.

| 감각 | 한자 | 독서에서의 의미 | 현재 대응 존 | 핵심 경험 |
|:----:|:----:|----------------|:------------:|-----------|
| 시(視) | 보다 | 문장의 아름다움을 눈으로 감상한다 | 필사 존 | 아름다운 문장을 필체·레이아웃과 함께 "보는" 독서 |
| 청(聽) | 듣다 | 목소리와 음악으로 책을 귀로 만난다 | 낭독 존 · 듣기 존 | 소리 내어 읽고, 음악과 함께 "듣는" 독서 |
| 촉(觸) | 쓰다 | 손끝으로 문장을 따라 쓰며 새긴다 | 필사 존 | 펜을 잡고 한 글자씩 따라 쓰는 "만지는" 독서 |
| 미(味) | 음미하다 | 깊이 곱씹어 자신의 언어로 표현한다 | 서평 존 | 책의 의미를 되새기고 글로 풀어내는 "맛보는" 독서 |
| 공(共) | 함께하다 | 타인과 생각을 나누며 공명한다 | 북클럽 존 | 토론하고 함께 읽으며 "나누는" 독서 |

### 2-2. 왜 "오감"인가 — 전통적 오감과의 차이

> 전통적 오감의 "후각·미각" 대신, 독서에 고유한 감각인 **"음미(味)"** 와 **"공명(共)"** 으로 재해석한 것이 오감도서관의 차별점입니다.

| 전통 오감 | 오감도서관 | 재해석 이유 |
|:---------:|:---------:|-------------|
| 시각 | **시(視)** | 그대로 유지. 문장·표지·필체 등 시각적 아름다움이 독서의 첫 번째 감각 |
| 청각 | **청(聽)** | 그대로 유지. 낭독의 목소리, 음악과의 감정 연결이 독서의 청각적 차원 |
| 촉각 | **촉(觸)** | 그대로 유지. 필사에서 펜을 잡고 쓰는 행위는 가장 원초적인 촉각 경험 |
| 후각 → 미(味) | **미(味)** | 후각은 독서에 직접 대응하기 어려움. 대신 "음미하다"는 깊이 곱씹는 사유의 감각으로 전환 |
| 미각 → 공(共) | **공(共)** | 미각은 독서에 직접 대응하기 어려움. 대신 "함께하다"는 공동체적 공명의 감각으로 전환 |

이 재해석은 오감도서관을 단순한 독서 플랫폼이 아닌, **"독서를 감각으로 체험하는 문화 공간"** 으로 포지셔닝합니다.

### 2-3. 오감과 5대 존의 관계

```
  ┌──────────────────────────────────────────────────┐
  │                   오감도서관                        │
  │                                                    │
  │   시(視)  ←───→  필사 존                            │
  │     └ 문장의 시각적 아름다움을 갤러리로 전시         │
  │                                                    │
  │   청(聽)  ←───→  낭독 존 + 듣기 존                  │
  │     └ 낭독: 나의 목소리로 책을 완성                  │
  │     └ 듣기: 음악의 감정으로 책을 만남                │
  │                                                    │
  │   촉(觸)  ←───→  필사 존                            │
  │     └ 손으로 쓰는 행위 자체가 독서                   │
  │                                                    │
  │   미(味)  ←───→  서평 존                            │
  │     └ 곱씹고 되새겨 자기 언어로 표현                │
  │                                                    │
  │   공(共)  ←───→  북클럽 존                          │
  │     └ 함께 읽고, 함께 토론하며 공명                 │
  └──────────────────────────────────────────────────┘
```

> **필사 존**은 시(視) + 촉(觸) 두 감각을 동시에 담당합니다.
> "아름다운 문장을 **보고(視)**, 손으로 **쓰며(觸)** 새긴다."

## 3. 5대 존(Zone) 기획 의도

### 3-1. 왜 "5대 존"인가

기존 독서 플랫폼들은 "책 목록 → 서평 작성"의 단선적 구조입니다. 오거서는 **독서 행위를 5가지 감각으로 분해**하여 학생들이 자신에게 맞는 방식으로 책에 접근하도록 설계했습니다.

학생마다 독서 성향이 다릅니다:
- "저는 조용히 혼자 읽고 서평 쓰는 게 좋아요" → **서평 존**
- "같이 읽고 떠들면 재밌잖아요" → **북클럽 존**
- "소리 내서 읽으면 집중이 잘 돼요" → **낭독 존**
- "좋은 문장 따라 쓰면 기분이 좋아요" → **필사 존**
- "음악 들으면서 책 읽는 게 취향이에요" → **듣기 존**

하나의 정답을 강요하지 않고, **다섯 가지 문을 모두 열어두는 것**이 오거서의 설계 철학입니다.

### 3-2. 존별 상세 기획 의도

#### 북클럽 존: "함께 읽고 토론하는 북클럽 존"

| 항목 | 내용 |
|------|------|
| **기획 목표** | 좌장(동문·교수) 중심의 세대 간 독서 대화를 유도하여, 독서가 "혼자 하는 것"이라는 인식을 깨뜨림 |
| **핵심 경험** | "모집 → 진행 → 토론"의 북클럽 생애주기를 시각화하여 **"살아있는 모임"** 임을 체감시킴 |
| **참여 유도 전략** | 모집중 펄스 뱃지 + 참여율 프로그레스 바 + D-day 마감 카운트로 "지금 신청하지 않으면 늦는다"는 긴박감 형성 |
| **색상** | 🟢 그린 (Green) — 새싹·성장·시작의 이미지. "함께 자라는" 독서 공동체를 상징 |
| **디자인 근거** | 피처 카드(대형) + 사이드 리스트 구조는 e-commerce의 "오늘의 특가 + 일반 상품" 패턴에서 차용. 가장 주목할 모임을 크게, 나머지는 리스트로 효율적 탐색 |
| **UX 흐름** | 메인(피처 카드) → "더 보기"(전체 목록) → 상세(좌장·일정·신청) |

#### 낭독 존: "소리 내어 완성하는 낭독 존"

| 항목 | 내용 |
|------|------|
| **기획 목표** | 묵독 중심 독서에서 벗어나 **"목소리로 책을 완성하는"** 협업 경험 제공. 한 권의 책을 여러 사람이 나누어 읽어 하나의 오디오북을 만드는 프로젝트 |
| **핵심 경험** | LIVE 상태 표시 + 마감 카운트다운으로 **지금 이 순간 진행되고 있는 활동**임을 강조. "나도 빈 파트를 맡아야겠다"는 참여 긴장감 형성 |
| **차별화 포인트** | 완성된 낭독은 아카이브에 보존되어 "우리가 함께 만든 오디오북"이라는 **성취감과 소유감** 제공 |
| **색상** | 🟣 퍼플 (Purple) — 낭독의 울림·깊이·교양의 이미지. 소리가 가진 고급스러운 느낌을 상징 |
| **디자인 근거** | 2컬럼 레이아웃(LIVE 카드 + 아카이브 리스트). 왼쪽의 LIVE 카드는 "지금 진행 중"이라는 시간성을, 오른쪽 아카이브는 "축적된 결과물"이라는 아카이브성을 동시에 전달 |
| **UX 흐름** | 메인(LIVE 카드 확인) → "참여하기" → 녹음 제출 → 아카이브에서 완성본 감상 |

#### 듣기 존: "듣고 읽어보는 듣기 존"

| 항목 | 내용 |
|------|------|
| **기획 목표** | 학생이 즐겨 듣는 노래의 감정을 AI로 분석하여 도서 발췌문과 매칭. **"내 감정에 맞는 책"** 이라는 새로운 큐레이션 경험 제공 |
| **핵심 경험** | "이 노래를 좋아하는 당신에게, 이 문장을 추천합니다." 음악이라는 익숙한 매개를 통해 책으로 자연스럽게 연결 |
| **차별화 포인트** | 기존 도서 추천이 "이 책을 읽은 사람이 이 책도 읽었습니다"(행동 기반)라면, 듣기 존은 **"이 감정을 느낀 당신에게 이 문장을"(감정 기반)** 으로 접근 |
| **색상** | 🔵 블루 (Blue) — 음악의 청량함·감정의 깊이. 소리와 물의 이미지를 공유하는 색상 |
| **디자인 근거** | 캐러셀 형식은 "음악 플레이리스트를 넘기듯" 콘텐츠를 탐색하는 경험. 상단에 노래 정보, 하단에 매칭 도서 발췌문을 배치하여 "이 노래 → 이 문장"의 인과를 시각적으로 전달 |
| **UX 흐름** | 캐러셀 탐색(노래-책 매칭 확인) → 감정 태그로 분위기 파악 → "더 보기"로 전체 플레이리스트 탐색 |

#### 필사 존: "따라 쓰며 새기는 필사 존"

| 항목 | 내용 |
|------|------|
| **기획 목표** | 좋은 문장을 손으로 옮겨 쓰는 행위 자체를 콘텐츠화. 갤러리 형태로 전시하여 **"나도 해보고 싶다"** 는 동기를 자극 |
| **핵심 경험** | 필사는 가장 "느린 독서"입니다. 한 글자 한 글자 따라 쓰며 문장의 의미를 천천히 새기는 행위. 이 느림 자체가 디지털 세대에게 **새로운 감각적 경험** |
| **이중 감각** | 필사 존은 **시(視) + 촉(觸)** 두 감각을 동시에 담당. 완성된 필사를 "보는" 즐거움(갤러리)과, 직접 "쓰는" 행위(필사 참여)를 모두 포함 |
| **색상** | 🟡 앰버 (Amber) — 오래된 양피지·잉크·따뜻한 조명의 이미지. 손으로 쓰는 행위의 아날로그적 따뜻함을 상징 |
| **디자인 근거** | 3컬럼 갤러리는 미술관 전시처럼 필사 결과물을 "작품"으로 대접. "이달의 필사" 뱃지는 큐레이션의 권위를 부여하여 참여 동기 강화 |
| **UX 흐름** | 갤러리 탐색(아름다운 문장 감상) → 상세(전문 + 작성자 코멘트) → "나도 필사해보기" 동기 형성 |

#### 서평 존: "읽고 생각을 나누는 서평 존"

| 항목 | 내용 |
|------|------|
| **기획 목표** | 단순 별점이 아닌 **심층 서평**을 유도. 우수 서평 선정과 카테고리 필터로 양질의 서평이 발견되는 구조 설계 |
| **핵심 경험** | "이 책을 이렇게 읽은 사람도 있구나." 서평은 자기 언어로 책을 재해석하는 행위. 타인의 서평을 읽으며 **같은 책의 다른 얼굴**을 발견 |
| **차별화 포인트** | 카테고리 필터(문학·인문·사회·과학·예술)로 관심 분야의 서평을 집중 탐색. 우수 서평 뱃지로 양질 콘텐츠에 **보상과 인정**의 동기 부여 |
| **색상** | 🔴 레드 (Red) — 열정·깊이·비평의 이미지. 서평이 가진 지적 에너지를 상징 |
| **디자인 근거** | 카테고리 필터 + 캐러셀 구조는 "넷플릭스 장르별 추천"에서 착안. 관심 장르를 선택하면 해당 분야 서평이 캐러셀로 펼쳐지는 직관적 탐색 경험 |
| **UX 흐름** | 카테고리 선택 → 캐러셀 탐색(별점·요약 미리보기) → 상세(전문 + 도서 정보 + 댓글) |

### 4-1. 구조 흐름: 관심 유도 → 탐색 → 깊이 있는 참여

메인페이지는 위에서 아래로 **3단계 흐름**을 따릅니다.

| 단계 | 영역 | 기획 의도 | 사용자 심리 |
|:----:|------|-----------|-------------|
| 1단계 | **하이라이트 배너** | 첫 화면에서 "지금 가장 주목할 활동"을 존별 그라데이션으로 시각적 임팩트 전달. 자동 재생으로 여러 존의 하이라이트를 자연스럽게 노출 | "뭔가 재밌는 게 있네?" (호기심 유발) |

| 3단계 | **존 섹션 (이맥스)** 또는 **피드 (인스타)** | 이맥스: 존별 최적화된 UI로 깊이 있는 정보 제공 / 인스타: 소셜 맥락의 가벼운 탐색 | "이것도 해보고 싶다" (참여 전환) |

## 5. 뷰 모드 전략

### 5-1. 왜 "두 가지 뷰 모드"인가

사용자 리서치 결과, 독서 플랫폼에 대한 기대가 크게 두 갈래로 나뉘었습니다.

| 유형 | 사용자 특성 | 기대 행동 | 제공 모드 |
|------|------------|-----------|-----------|
| 탐색형 | "북클럽 신청하려고 왔어요" | 특정 존에 집중하여 정보를 깊이 탐색 | **이맥스 모드** — 존별 섹션 순차 배치, 각 존의 핵심 지표(참여율, 별점, 마감일 등)를 전용 UI로 표현 |
| 브라우징형 | "뭐 올라왔나 볼게요" | SNS처럼 전체 활동을 빠르게 훑기 | **인스타 모드** — 시간순 통합 피드, 무한 스크롤, 댓글 미리보기로 소셜 맥락 제공 |

두 모드는 상단 pill 토글 하나로 즉시 전환되며, 같은 콘텐츠를 **목적에 맞는 다른 형태**로 보여줍니다.

### 5-2. 모드별 상세 비교

| 항목 | 이맥스 모드 | 인스타 모드 |
|------|------------|------------|
| 레이아웃 | 존별 섹션 (카드 그리드 + 캐러셀) | 단일 타임라인 피드 |
| 탐색 방식 | 존 단위 스크롤 → 더 보기 | 무한 스크롤 (시간순) |
| 콘텐츠 표시 | 존별 최적화 UI (프로그레스 바, 별점 등) | 통일된 카드 포맷 |
| 상호작용 | 존 페이지 이동 중심 | 좋아요 · 댓글 미리보기 · 상세 링크 |
| 데이터 소스 | 존별 개별 API 5개 | 통합 Feed API 1개 |
| 대상 사용자 | 특정 존 집중 탐색 | 전체 활동 빠르게 훑기 |

### 5-3. 인스타 모드 카드 구조

```
┌─────────────────────────┐
│ 🟢 김성균 · 국문학과  [필사]│  ← 아바타 + 이름 + 존 배지
├─────────────────────────┤
│                         │
│   "나는 나비처럼 자유롭게  │  ← 콘텐츠 (존 타입별 다름)

│                         │
├─────────────────────────┤
│ ♥ 12   💬 3   · 2시간 전 │  ← 좋아요 + 댓글수 + 시간
```

## 6. 상세 페이지 기획 의도

### 6-1. 상세 페이지의 역할

메인페이지가 "관심 유도"라면, 상세 페이지는 **"참여 전환"** 의 공간입니다. 사용자가 메인에서 흥미를 느꼈을 때, 상세 페이지에서 충분한 정보와 참여 수단을 제공하여 **"보기만 하는 사람"을 "참여하는 사람"으로** 전환합니다.

### 6-2. 페이지별 상세 기획

#### 북클럽 상세 (`/bookclub/[id]`)

| 항목 | 기획 의도 |
|------|-----------|
| **좌장 프로필** | 좌장의 이름·학과·학번·경력·소개를 상세히 공개하여 **"누가 이끄는 모임인지"** 투명하게 전달. 좌장의 권위가 참여 결정의 가장 큰 요인 |
| **도서 정보 카드** | 선정 도서의 제목·저자·출판사·카테고리·소개문을 카드 형태로 표시. 학생이 **"이 책을 읽어볼 가치가 있는지"** 즉시 판단할 수 있도록 정보 제공 |
| **일정 정보** | 기간·토론일·장소를 아이콘(달력·시계·핀)과 함께 표시. "시간이 되는지"를 한눈에 확인 가능 |
| **참여 현황 프로그레스 바** | 현재 인원 / 정원을 프로그레스 바 + 퍼센트로 시각화. "자리가 얼마 안 남았다"는 **긴박감** 전달 |
| **상태별 CTA** | 모집중 → "북클럽 신청하기" 버튼 / 진행중·완료 → 버튼 비노출. 행동 가능한 상태에서만 CTA를 보여 혼란 방지 |

#### 필사 상세 (`/writing/[id]`)

| 항목 | 기획 의도 |
|------|-----------|
| **발췌문 blockquote** | 필사한 문장을 좌측 앰버색 보더 blockquote로 크게 배치. **문장 자체의 힘**을 느끼게 하는 것이 핵심. 폰트 크기를 키워 "작품"으로 대접 |
| **도서 출처** | 필사 원문의 도서명·저자를 표시하여 "이 문장이 어디서 왔는지"를 알림. 원전으로의 역추적 경로 제공 |
| **작성자 코멘트** | 필사자가 남긴 한마디. "왜 이 문장을 필사했는지"의 맥락을 제공하여 댓글 대화의 시작점이 됨 |
| **좋아요 + 댓글** | "이 문장이 왜 좋은지"의 대화를 유도. 좋아요는 공감의 표현, 댓글은 감상의 나눔 |
| **이달의 필사 뱃지** | 운영팀이 선정한 우수 필사에 부여. 참여자에게 **인정과 보상**의 동기 제공 |

#### 서평 상세 (`/review/[id]`)

| 항목 | 기획 의도 |
|------|-----------|
| **별점 → 한줄 요약 → 본문** 배치 순서 | 서평의 깊이를 **단계적으로 경험**. 별점(전체 인상) → 한줄 요약(핵심 메시지) → 본문(상세 논의) 순으로 읽으며 점점 깊이 들어감 |
| **도서 정보 카드** | 서평이 어떤 책에 대한 것인지를 독립 카드로 표시. "이 서평이 좋으면 이 책을 읽어보자"는 행동 전환 유도 |
| **우수 서평 뱃지** | 트로피 아이콘으로 시각적 차별화. 양질 콘텐츠에 대한 **보상 체계**를 가시화하여 서평 작성 동기 강화 |
| **좋아요 + 댓글** | 서평에 대한 공감과 토론. "나는 다르게 읽었다"는 반론도 환영하는 열린 구조 |

## 7. 디자인 원칙

### 7-1. 핵심 원칙 5가지

| 원칙 | 적용 방식 | 기획 의도 |
|------|-----------|-----------|
| **존 = 색상** | 북클럽(🟢그린)·낭독(🟣퍼플)·듣기(🔵블루)·필사(🟡앰버)·서평(🔴레드) | 색상만으로 어떤 존인지 즉시 인지. 존의 정체성을 시각적으로 각인 |
| **진행 상태의 시각화** | 모집중 펄스 뱃지, D-day 카운트다운, 참여율 프로그레스 바 | 정적 콘텐츠가 아닌 **"살아있는 활동"** 임을 강조. 참여를 유도하는 긴장감 형성 |
| **행동 유도(CTA) 최소화** | 페이지당 핵심 CTA 1개 (신청하기, 좋아요, 댓글 작성) | 선택지를 줄여 전환율 향상. "뭘 해야 하지?"가 아닌 "이걸 하면 되겠다" |
| **스켈레톤 우선 로딩** | 모든 섹션에 animate-pulse 스켈레톤 적용 | 빈 화면 대신 레이아웃을 먼저 보여줘 체감 속도 향상. 사용자가 이탈하지 않고 기다림 |
| **점진적 깊이** | 메인(미리보기) → 존 목록(리스트) → 상세(전문) | 3단계로 정보의 깊이를 조절. 사용자가 원하는 만큼만 깊이 들어갈 수 있는 자율적 구조 |

| # | 제안 | 설명 | 기대 효과 | 기획 의도 |
|---|------|------|-----------|-----------|
| V1 | 필사 이미지 실사 업로드 | 손글씨 촬영 이미지를 직접 업로드하는 기능. 갤러리가 실제 필체로 채워지면 시각적 다양성과 개성이 살아남 | 필사 참여 동기 부여, "내 글씨도 올리고 싶다" | 필사의 "시각적 아름다움"을 복원. 타이핑이 아닌 **손글씨**가 갤러리에 올라갈 때 비로소 시(視)의 감각이 완성됨 |

| # | 제안 | 설명 | 기대 효과 | 기획 의도 |
|---|------|------|-----------|-----------|
| A1 | 낭독 인라인 오디오 플레이어 | 낭독 상세 페이지에 웹 오디오 플레이어 내장. 파형(waveform) 시각화로 소리를 "보이게" 표현 | 낭독 존의 핵심 경험 완성 | 낭독 존의 본질은 **"소리"**. 텍스트만으로는 낭독의 감동을 전달할 수 없음. 오디오 플레이어가 있어야 비로소 청(聽)의 존이 됨 |
| A2 | 아카이브 연속 재생 | 과거 낭독 아카이브를 플레이리스트처럼 연속 재생. "이달의 오디오북"을 통근·산책 중 청취 | 재방문율·체류 시간 증가 | 낭독 아카이브의 가치를 극대화. **"우리가 만든 오디오북을 들으며 출퇴근한다"** — 참여의 결과물이 일상에 스며드는 경험 |
| A3 | 듣기 존 음원 연동 | YouTube/Spotify 임베드 또는 30초 미리듣기로 노래를 실제로 들으며 매칭 도서 확인 | 감정-도서 매칭의 설득력 강화 | 듣기 존의 핵심은 **"음악의 감정과 도서의 감정이 공명한다"**는 것. 실제로 노래를 들어야 그 공명을 체감할 수 있음 |
| A4 | 독서 ASMR / 앰비언스 | 페이지 넘기는 소리, 카페 배경음, 빗소리 등 독서 분위기 배경음 제공 | 플랫폼 자체가 "독서 공간"이 되는 경험 | 오감도서관을 **"소리로 둘러싸인 독서 공간"**으로 승격. 디지털에서도 독서의 물리적 감각을 재현 |

#### 촉(觸) — "쓰는 독서" 강화

| # | 제안 | 설명 | 기대 효과 | 기획 의도 |
|---|------|------|-----------|-----------|
| T1 | 디지털 필사 캔버스 | 태블릿·마우스로 직접 손글씨를 쓸 수 있는 인앱 캔버스. 작성 과정을 타임랩스 영상으로 저장 | 오프라인 필사를 디지털로 전환, 진입 장벽 하락 | 필사의 본질은 **"쓰는 행위"** 자체. 현재는 결과물만 보여주고 있으나, 쓰는 과정을 디지털로 체험하게 하면 촉(觸)의 감각이 완성됨 |
| T2 | 필사 챌린지 캘린더 | 매일 1문장 필사를 30일간 이어가는 캘린더 챌린지. 연속 달성 시 뱃지 + 포인트 | 습관 형성, 일일 재방문 유도 | 필사를 **"일회성 행사"가 아닌 "일상 습관"**으로 전환하기 위한 게이미피케이션. 매일의 작은 촉각 경험이 쌓여 독서 습관이 됨 |
| T3 | 필사 도구 추천 | 펜·노트·잉크 등 필사 도구를 추천하는 큐레이션 섹션. 학술정보관 내 필사 코너와 연계 | 온라인→오프라인 연결, 촉각 경험의 실체화 | 디지털 필사를 넘어 **실제 펜을 잡는 경험**으로 이끄는 O2O(Online to Offline) 전략. 학술정보관 방문 유도 |

#### 미(味) — "음미하는 독서" 강화

| # | 제안 | 설명 | 기대 효과 | 기획 의도 |
|---|------|------|-----------|-----------|
| M1 | 서평 토론 스레드 | 서평 하단에 찬성/반대 의견을 트리 구조로 달 수 있는 토론 기능. 서평이 대화의 시작점이 됨 | 단방향 서평 → 양방향 토론으로 깊이 확장 | 음미의 본질은 **"곱씹기"**. 한 번 쓴 서평에 반론이 달리고, 그에 대한 재반론이 이어지며 **독서의 깊이가 확장**되는 구조 |
| M2 | 독서 노트 기능 | 책을 읽으며 챕터별 메모·밑줄·감상을 기록하는 개인 독서 노트. 선택적으로 공개 가능 | 서평 이전 단계의 "음미" 과정을 구조화 | 서평은 "완성된 감상"이지만, 음미는 **"읽으면서 느끼는 과정"**. 독서 노트는 그 과정을 기록하는 도구. 과정이 있어야 결과(서평)도 깊어짐 |
| M3 | 도서-음료 페어링 | "이 책을 읽을 때 어울리는 음료"를 추천하는 경험 요소. 학내 카페와 콜라보 가능 | 독서를 미각과 연결하는 감각적 브랜딩 | 전통적 오감의 "미각"을 살려 **문자 그대로의 오감** 경험을 완성하는 유쾌한 실험. 커피 한 잔과 책 한 권의 조합이라는 로맨틱한 이미지 |

#### 공(共) — "함께하는 독서" 강화

| # | 제안 | 설명 | 기대 효과 | 기획 의도 |
|---|------|------|-----------|-----------|
| C1 | 토론 기록 아카이브 | 북클럽 토론 후 핵심 논점·참여자 발언 요약을 기록하는 회의록 기능 | 모임의 지적 산출물이 축적되는 구조 | 북클럽의 가치는 **"토론에서 나온 통찰"**. 현재는 모임 자체만 기록되고 토론 내용은 사라짐. 아카이브가 있어야 "함께 만든 지적 유산"이 남음 |
| C2 | 실시간 함께 읽기 | 같은 시간에 같은 책을 읽는 사람들끼리 "조용한 독서방"에 접속. 온라인 독서실 컨셉 | 혼자이지만 함께라는 연대감 | **"옆에서 같이 읽는 느낌"**을 디지털로 구현. 도서관에서 나란히 앉아 각자의 책을 읽는 경험. 공(共)의 가장 순수한 형태 |
| C3 | 존 간 크로스오버 | 같은 도서에 대한 필사·서평·낭독을 하나의 "도서 허브" 페이지로 통합 조회 | 한 권의 책을 다섯 감각으로 체험한 전체 그림 | 오감도서관의 궁극적 완성. **하나의 책을 시(필사)·청(낭독)·촉(필사)·미(서평)·공(북클럽) 다섯 감각으로 체험한 결과가 한곳에 모임** |
| C4 | LC(학습공동체) 대시보드 | LC 단위로 멤버들의 5존 활동을 종합 시각화. 존별 참여율 레이더 차트 | 팀 단위 독서 목표 관리, 선의의 경쟁 유도 | 개인 독서를 넘어 **팀 독서**로 확장. "우리 LC는 낭독은 잘하지만 서평이 부족해" → 팀 단위의 오감 밸런스를 맞추는 게임적 경험 |

## 13. 결론

### 오감도서관이 그리는 미래

오거서 메인페이지 리팩토링은 단순한 UI 개선이 아닙니다.

**8개 혼합 섹션 → 5대 존 중심 구조**로의 전환은, 독서 행위를 **다섯 가지 감각으로 분해하고 재조합하는 철학적 전환**입니다.

| 단계 | 의미 |
|------|------|
| 현재 | 5대 존의 구조를 확립하고, 각 존의 핵심 경험을 디지털 UI로 구현 |
| Phase 1~2 | 각 감각의 "핵심 경험"을 완성 (소리를 듣고, 이미지를 보고, 토론을 기록하는) |
| Phase 3~4 | 감각 간 크로스오버로 "오감 통합 독서 경험" 완성 |
| 궁극적 목표 | 한 권의 책을 시(視)·청(聽)·촉(觸)·미(味)·공(共) — **온몸으로 체험하는 독서 플랫폼** |

### 핵심 수치

- 이맥스 모드: 각 존의 핵심 콘텐츠를 emax 디자인 패턴으로 명확하게 전달
- 인스타 모드: SNS 친화적 타임라인으로 전체 활동을 한눈에 파악
- 상세 페이지 3종: 북클럽·필사·서평의 개별 콘텐츠까지 완결된 사용자 경험 제공
- 총 **17개 파일 신규 생성 (2,070 lines), 1개 파일 수정**, 빌드 에러 **0건**
