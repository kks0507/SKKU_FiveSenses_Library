# 오거서(五去書) 메인페이지 리팩토링 최종 보고서

> **작성일:** 2026.02.09
> **대상 시스템:** 성균관대학교 학술정보관 독서 플랫폼 "오거서"
> **기술 스택:** Next.js 16 / React 19 / TypeScript / Tailwind CSS 4 / Swiper 12

---

## 1. 개요

기존 메인페이지는 8개 섹션(HeroBanner, ZoneGrid, LiveActivity, Bookclub, Campaign, RecommendedBooks, Ranking, Services)이 나열되어 **정보 과부하 및 존(Zone) 정체성 부재** 문제가 있었습니다.

이번 리팩토링은 다음 3가지 목표를 달성합니다.

| # | 목표 | 해결 방안 |
|---|------|-----------|
| 1 | 5대 존 중심 구조 확립 | 북클럽 · 낭독 · 듣기 · 필사 · 서평 — 존별 독립 섹션 배치 |
| 2 | 두 가지 뷰 모드 제공 | **이맥스 모드**(카드 기반) + **인스타 모드**(타임라인 피드) 토글 |
| 3 | 상세 페이지 완성 | 북클럽 · 필사 · 서평 상세 페이지 3종 신규 구현 |

---

## 2. 기획 의도

### 2-1. 플랫폼 비전

오거서(五去書)는 **"다섯 가지 방법으로 책에 다가간다"** 는 뜻을 담은 성균관대학교 학술정보관의 독서 진흥 플랫폼입니다.

| 항목 | 내용 |
|------|------|
| 미션 | 대학생의 독서 습관 형성과 독서 공동체 활성화 |
| 핵심 가치 | 혼자가 아닌 **함께 읽는 경험**, 읽기를 넘어 **표현하는 독서** |
| 대상 사용자 | 성균관대학교 재학생 · 교직원 · 동문 |
| 운영 주체 | 학술정보관 독서문화팀 |

### 2-2. 왜 "5대 존" 인가

기존 독서 플랫폼들은 "책 목록 → 서평 작성"의 단선적 구조입니다. 오거서는 **독서 행위를 5가지 감각으로 분해**하여 학생들이 자신에게 맞는 방식으로 책에 접근하도록 설계했습니다.

| 존 | 독서 행위 | 기획 의도 |
|----|-----------|-----------|
| 북클럽 | 함께 읽고 토론한다 | 좌장(동문·교수) 중심의 세대 간 독서 대화를 유도. 모집→진행→토론의 생애주기를 시각화하여 "살아있는 모임"임을 체감시킴 |
| 낭독 | 소리 내어 읽는다 | 묵독 중심 독서에서 벗어나 "목소리로 책을 완성하는" 협업 경험 제공. LIVE 상태와 마감 카운트다운으로 참여 긴장감 형성 |
| 듣기 | 감정으로 책을 만난다 | 학생이 즐겨 듣는 노래의 감정을 AI로 분석하여 도서 발췌문과 매칭. "내 감정에 맞는 책"이라는 새로운 큐레이션 경험 |
| 필사 | 따라 쓰며 새긴다 | 좋은 문장을 손으로 옮겨 쓰는 행위 자체를 콘텐츠화. 갤러리 형태로 전시하여 "나도 해보고 싶다"는 동기를 자극 |
| 서평 | 읽고 생각을 나눈다 | 단순 별점이 아닌 심층 서평을 유도. 우수 서평 선정과 카테고리 필터로 양질의 서평이 발견되는 구조 설계 |

### 2-3. 왜 "두 가지 뷰 모드" 인가

사용자 리서치 결과, 독서 플랫폼에 대한 기대가 크게 두 갈래로 나뉘었습니다.

| 유형 | 사용자 특성 | 기대 행동 | 제공 모드 |
|------|------------|-----------|-----------|
| 탐색형 | "북클럽 신청하려고 왔어요" | 특정 존에 집중하여 정보를 깊이 탐색 | **이맥스 모드** — 존별 섹션 순차 배치, 각 존의 핵심 지표(참여율, 별점, 마감일 등)를 전용 UI로 표현 |
| 브라우징형 | "뭐 올라왔나 볼게요" | SNS처럼 전체 활동을 빠르게 훑기 | **인스타 모드** — 시간순 통합 피드, 무한 스크롤, 댓글 미리보기로 소셜 맥락 제공 |

두 모드는 상단 pill 토글 하나로 즉시 전환되며, 같은 콘텐츠를 **목적에 맞는 다른 형태**로 보여줍니다.

### 2-4. 메인페이지 구조 기획

메인페이지는 위에서 아래로 **관심 유도 → 탐색 → 깊이 있는 참여** 의 3단계 흐름을 따릅니다.

| 단계 | 영역 | 기획 의도 |
|:----:|------|-----------|
| 1단계 | **하이라이트 배너** | 첫 화면에서 "지금 가장 주목할 활동"을 존별 그라데이션으로 시각적 임팩트 전달. 자동 재생으로 여러 존의 하이라이트를 자연스럽게 노출 |
| 2단계 | **뷰 모드 토글** | 사용자에게 탐색 방식의 선택권을 부여. "나에게 맞는 방식으로 보세요"라는 메시지 |
| 3단계 | **존 섹션 (이맥스)** 또는 **피드 (인스타)** | 이맥스: 존별 최적화된 UI로 깊이 있는 정보 제공 / 인스타: 소셜 맥락의 가벼운 탐색 |

### 2-5. 상세 페이지 기획 의도

| 상세 페이지 | 기획 의도 |
|------------|-----------|
| 북클럽 상세 | 좌장의 권위(프로필·경력)와 모임의 실체(일정·장소·인원)를 투명하게 공개하여 **신뢰 기반 참여 유도**. 프로그레스 바로 "자리가 얼마 안 남았다"는 긴박감 전달 |
| 필사 상세 | 발췌문을 blockquote로 크게 배치하여 **문장 자체의 힘**을 느끼게 함. 작성자 코멘트와 댓글로 "이 문장이 왜 좋은지"의 대화를 유도 |
| 서평 상세 | 별점 → 한줄 요약 → 본문 → 도서 정보 순으로 배치하여 **서평의 깊이를 단계적으로 경험**. 우수 서평 뱃지로 양질 콘텐츠 보상 체계 마련 |

### 2-6. 디자인 원칙

| 원칙 | 적용 방식 |
|------|-----------|
| **존 = 색상** | 북클럽(그린)·낭독(퍼플)·듣기(블루)·필사(앰버)·서평(레드) — 색상만으로 어떤 존인지 즉시 인지 |
| **진행 상태의 시각화** | 모집중 펄스 뱃지, D-day 카운트다운, 참여율 프로그레스 바 — 정적 콘텐츠가 아닌 "살아있는 활동"임을 강조 |
| **행동 유도(CTA) 최소화** | 페이지당 핵심 CTA 1개 (신청하기, 좋아요, 댓글 작성) — 선택지를 줄여 전환율 향상 |
| **스켈레톤 우선 로딩** | 모든 섹션에 스켈레톤 적용 — 빈 화면 대신 레이아웃을 먼저 보여줘 체감 속도 향상 |
| **점진적 깊이** | 메인(미리보기) → 존 목록(리스트) → 상세(전문) — 3단계로 정보의 깊이를 조절 |

## 3. 기능 설명

### 3-1. 메인페이지 공통 기능

| 기능 | 설명 | 동작 방식 |
|------|------|-----------|
| 하이라이트 배너 | 각 존의 주요 콘텐츠를 존별 그라데이션 배경으로 소개 | 5초 간격 자동 재생, 하단 컨트롤(일시정지/재생·슬라이드 이동), 클릭 시 상세 페이지 이동 |
| 뷰 모드 토글 | "이맥스" / "인스타" 두 가지 보기 방식 전환 | pill 형태 버튼 클릭으로 즉시 전환, 선택 상태는 색상으로 표시 |

### 3-2. 이맥스 모드 — 5대 존 기능

#### 북클럽 존: 함께 읽고 토론하는 북클럽 존

| 기능 | 설명 |
|------|------|
| 피처 카드 | 현재 모집/진행 중인 북클럽을 대형 카드로 노출. 선정 도서, 좌장 이름, 모집 상태 뱃지 표시 |
| 참여 현황 바 | 현재 인원 / 정원을 프로그레스 바 + 퍼센트로 시각화 |
| 모집 상태 표시 | 모집중(초록 펄스) · 진행중(파랑) · 완료(회색) 뱃지 자동 적용 |
| 신청하기 CTA | 모집중인 북클럽에 한해 "신청하기" 버튼 노출 |
| 사이드 목록 | 나머지 북클럽을 소형 카드 리스트로 표시, 클릭 시 상세 이동 |
| 더 보기 | 우측 상단 "더 보기" 링크 → `/bookclub` 목록 페이지 이동 |

#### 낭독 존: 소리 내어 완성하는 낭독 존

| 기능 | 설명 |
|------|------|
| LIVE 낭독 카드 | 이달의 낭독 과제를 LIVE 뱃지 + 마감 D-day 카운트다운과 함께 표시 |
| 과제 정보 | 낭독 대상 도서명, 해당 장·페이지 범위, 과제 설명 표시 |
| 참여 현황 바 | 현재 참여자 / 전체 목표 인원을 프로그레스 바로 시각화 |
| 아카이브 목록 | 과거 완성된 낭독 오디오북 목록, 재생 시간(분:초) 표시 |
| 더 보기 | 우측 상단 "더 보기" 링크 → `/narration` 페이지 이동 |

#### 듣기 존: 듣고 읽어보는 듣기 존

| 기능 | 설명 |
|------|------|
| 노래-도서 매칭 캐러셀 | 사용자가 들은 노래와 감정 분석으로 매칭된 도서 발췌문을 Swiper 캐러셀로 표시 |
| 감정 태그 | 각 매칭 카드에 분위기(mood) 뱃지 + 감정 키워드 태그 표시 |
| 좌우 화살표 | 데스크톱에서 캐러셀 좌우 화살표 버튼으로 탐색 |
| 반응형 슬라이드 | 모바일 1장 → 태블릿 2~3장 → 데스크톱 4장 자동 조절 |
| 더 보기 | 우측 상단 "더 보기" 링크 → `/listening` 페이지 이동 |

#### 필사 존: 따라 쓰며 새기는 필사 존

| 기능 | 설명 |
|------|------|
| 3컬럼 갤러리 | 최신 필사 6건을 반응형 그리드(모바일 1열·태블릿 2열·데스크톱 3열)로 표시 |
| 발췌문 미리보기 | 각 카드에 필사한 문장을 인용 형태로 2줄까지 표시 |
| 이달의 필사 뱃지 | 운영팀이 선정한 이달의 필사에 앰버색 뱃지 표시 |
| 호버 확대 | 카드 위에 마우스를 올리면 아이콘 확대(scale-110) 애니메이션 |
| 좋아요 수 | 각 카드 하단에 하트 아이콘 + 좋아요 수 표시 |
| 더 보기 | 우측 상단 "더 보기" 링크 → `/writing` 페이지 이동 |

#### 서평 존: 읽고 생각을 나누는 서평 존

| 기능 | 설명 |
|------|------|
| 카테고리 필터 | 전체·문학·인문·사회·과학·예술 6개 탭으로 서평 필터링 |
| 서평 캐러셀 | 필터링된 서평을 Swiper 캐러셀로 표시, 좌우 화살표 탐색 |
| 별점 표시 | 각 카드에 1~5점 별점(StarRating 컴포넌트) 표시 |
| 우수 서평 뱃지 | 운영팀 선정 우수 서평에 트로피 아이콘 뱃지 표시 |
| 요약 인용 | 서평 요약문을 이탤릭 인용 형태로 2줄까지 미리보기 |
| 더 보기 | 우측 상단 "더 보기" 링크 → `/review` 페이지 이동 |

### 3-3. 인스타 모드 기능

| 기능 | 설명 |
|------|------|
| 통합 타임라인 | 필사·서평·낭독·북클럽 콘텐츠를 최신순으로 혼합 표시 |
| 무한 스크롤 | 하단 도달 시 자동으로 다음 10건 로딩 (IntersectionObserver) |
| 존 배지 | 각 카드 우측 상단에 콘텐츠 유형별 색상 뱃지(필사=앰버, 서평=레드, 낭독=퍼플, 북클럽=그린) |
| 사용자 아바타 | 이름 첫 글자를 존 색상 원형 아바타로 표시, 이름·학과 함께 노출 |
| 존별 콘텐츠 카드 | 필사=발췌문 인용박스, 서평=제목+별점+요약, 낭독=구간 정보, 북클럽=소개문 |
| 좋아요·댓글 수 | 각 카드 하단에 하트·말풍선 아이콘과 수치, 상대 시간(2시간 전 등) 표시 |
| 댓글 미리보기 | 최근 댓글 2건을 카드 내에 바로 표시 |
| 상세 링크 | "댓글 N개 모두 보기" + "자세히 보기" 클릭 시 각 상세 페이지 이동 |

### 3-4. 상세 페이지 기능

#### 북클럽 상세 (`/bookclub/[id]`)

| 기능 | 설명 |
|------|------|
| 도서 정보 카드 | 선정 도서의 제목·저자·출판사·카테고리·소개문을 카드 형태로 표시 |
| 좌장 프로필 | 좌장의 이름·학과·학번·경력·소개를 아바타와 함께 표시 |
| 일정 정보 | 진행 기간, 토론일, 장소를 아이콘(달력·시계·핀)과 함께 표시 |
| 참여 현황 바 | 현재 인원 / 정원을 프로그레스 바 + 퍼센트로 시각화 |
| 모집 상태 뱃지 | 모집중(초록 펄스) · 진행중(파랑) · 완료(회색) |
| 신청하기 CTA | 모집중 상태일 때만 하단에 전체 너비 "북클럽 신청하기" 버튼 표시 |
| 뒤로가기 | 좌측 상단 화살표 + "북클럽 목록" 텍스트 링크 |
| 로딩 스켈레톤 | 데이터 로딩 중 애니메이션 스켈레톤 UI 표시 |
| 404 처리 | 존재하지 않는 ID 접근 시 안내 메시지 + 목록 복귀 링크 |

#### 필사 상세 (`/writing/[id]`)

| 기능 | 설명 |
|------|------|
| 발췌문 표시 | 필사한 문장을 좌측 앰버색 보더 blockquote로 강조 표시 |
| 도서 출처 | 필사 원문의 도서명·저자 표시 |

## 9. "오감도서관" 컨셉 확장 제안

### 9-1. 컨셉 정의

오거서의 "다섯 가지 방법으로 책에 다가간다"는 철학을 **오감(五感)** 프레임으로 확장하면, 독서 행위가 단순한 정보 소비가 아닌 **온몸으로 책을 체험하는 감각적 경험**이 됩니다.

| 감각 | 한자 | 독서에서의 의미 | 현재 대응 존 |
|:----:|:----:|----------------|:------------:|
| 시(視) | 보다 | 문장의 아름다움을 눈으로 감상한다 | 필사 존 |
| 청(聽) | 듣다 | 목소리와 음악으로 책을 귀로 만난다 | 낭독 존 · 듣기 존 |
| 촉(觸) | 쓰다 | 손끝으로 문장을 따라 쓰며 새긴다 | 필사 존 |
| 미(味) | 음미하다 | 깊이 곱씹어 자신의 언어로 표현한다 | 서평 존 |
| 공(共) | 함께하다 | 타인과 생각을 나누며 공명한다 | 북클럽 존 |

> 전통적 오감의 "후각·미각" 대신, 독서에 고유한 감각인 **"음미(味)"** 와 **"공명(共)"** 으로 재해석한 것이 오감도서관의 차별점입니다.

### 9-2. 현황 진단 — 감각별 구현 수준

| 감각 | 존 | 현재 구현 수준 | 진단 |
|:----:|-----|:-----------:|------|
| 시(視) | 필사 | ★★☆☆☆ | 갤러리 구조는 갖췄으나 실제 손글씨 이미지가 placeholder. 시각적 몰입감 부족 |
| 청(聽) | 낭독 | ★★★☆☆ | LIVE 뱃지·참여 현황은 우수하나 실제 오디오 재생이 불가. 소리의 존인데 소리가 없음 |
| 청(聽) | 듣기 | ★★★☆☆ | 노래-책 매칭 로직 우수. 다만 실제 음원 미연동으로 "듣기" 경험이 텍스트에 그침 |
| 촉(觸) | 필사 | ★★☆☆☆ | 결과물 전시에 집중. 필사하는 "행위" 자체를 유도하는 장치 부족 |
